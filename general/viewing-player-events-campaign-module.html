Le module Campaign permet de suivre les événements de visualisation. La fréquence à laquelle les événements de vue sont envoyés au module Campaign est calculée dynamiquement en fonction de la durée de la vidéo. La campagne enverra au moins 4 événements, un par quartile.

Remarque: Les données de l'événement View peuvent être exportées à l'aide de l'option Exporter les données ou de l'API Audience.

Pour afficher les événements de la vue, ouvrez le module Campaign et cliquez sur All Recent Activity (Toutes les activités récentes) dans le volet La page affichera l'activité de l'événement de visualisation des dernières 24 heures.

Le lien Afficher les événements d'interaction sera activé si des événements ou des mesures personnalisés sont associés à un événement de vue.

La page affiche les informations suivantes.

  • Timestamp: date et heure de visionnage de la vidéo
  • Nom et ID de la vidéo: nom de la vidéo et identifiant vidéo Video Cloud
  • Pourcentage de visionnage: point auquel le spectateur a arrêté de regarder la vidéo, divisé par la durée de la vidéo. Par exemple, si un téléspectateur regardait les 5 premières secondes d'une vidéo de deux minutes, puis passait au point 55 secondes, regardait 5 secondes de plus et arrêtait la vidéo (ou quittait la page), le pourcentage regardé serait enregistré à 50 %.
    Remarque: Pour les vidéos en direct, il n'est pas possible de suivre le pourcentage de visionnage dans Campaign. Comme il n'est pas possible de savoir combien de temps il durera, le module Campaign ne suivra que le temps total regardé.
  • Durée de visionnage: durée totale de visionnage
  • ID externe: valeur utilisée pour identifier le spectateur, par exemple le GUID Eloqua, le GUID Marketo, le GUID HubSpot, le GUID SSO ou toute autre valeur unique utilisée pour identifier le spectateur.
  • Status: statut de l'événement
    • non traité - l'événement n'a pas été synchronisé avec la plateforme d'automatisation du marketing
    • synchronisé - l'événement a été synchronisé avec succès avec la plateforme d'automatisation du marketing
    • sync_disabled - Le module de campagne a été configuré avec la synchronisation désactivée, de sorte que l'événement a été ignoré
    • failed - erreur de synchronisation ; l'info-bulle fournit des détails sur l'erreur qui s'est produite
    • unknown_contact - utilisateur inconnu donc l'événement n'a pas été synchronisé
    • N/A - non applicable ; affiché lors de l'utilisation des intégrations Custom, Campaign Tracking ou Marketo Munchkin

Les données d'événement peuvent également être filtrées par statut à l'aide de la liste déroulante en haut de la page.

Affichage des événements d'interaction

Si des mesures ou des événements personnalisés sont associés à un événement de vue, le lien Afficher les événements d'interaction sera activé. Par exemple, si vous utilisez une expérience Gallery In-Page avec un lecteur compatible avec la campagne et que l'expérience est configurée avec des liens, des cartes ou tout autre type d'élément interactif, l'expérience enverra des événements d'impression et de conversion à la campagne module. Ces événements sont transmis à Campaign à l'aide des champs personnalisés Campaign. Les données de l'événement apparaîtront dans le module Campagne dans le cadre de toutes les activités récentes.

Remarque: Les événements d'interaction peuvent également apparaître lors de l'utilisation des indicateurs et événements personnalisés de Campaign.

Les données suivantes peuvent être transmises:

  • Événement 1 - Liste séparée par des virgules de noms de CTA avec lesquels le spectateur a eu la chance d'interagir (impression)
  • Événement 2 - Liste séparée par des virgules des noms de CTA avec lesquels le spectateur a interagi (conversion)
  • Métrique 1: nombre total d'impressions CTA
  • Mesure 2: nombre total de conversions CTA, c'est-à-dire lien cliqué, envoi de formulaire, etc.

Pour afficher les événements, cliquez sur le lien Afficher les événements d'interaction dans l'en-tête.

Autres options d'affichage des données

Les données de campagne peuvent être visualisées/exportées de différentes manières:

  • Activité vidéo: permet de visualiser l'activité des spectateurs par vidéo. Pour plus d'informations, voir Affichage de l'activité vidéo.
  • Profils de visionneuse: permet de visualiser l'activité vidéo par spectateur. Pour plus d'informations, consultez la section Affichage des informations sur le profil
  • Exporter les données - Utilisé pour exporter les données de visionnage de vidéos et de prospects. Pour plus d'informations, voir Exportation des données de campagne à partir du module Campaign.
  • API Audience - L'API Audience peut être utilisée pour récupérer par programme les données de visionnage de vidéos et de prospects. Pour plus d'informations, voir Vue d'ensemble: API Audience.

Foire aux questions

Comment les événements View sont-ils synchronisés avec les connexions MAP tierces ?
Vous pouvez trouver plusieurs enregistrements pour la même session de lecteur vidéo. Campaign considère un nouvel Afficher l'événement-fr/ si nous ne recevons pas d'activité après 15 minutes, même si l'ID de session est le même. De plus, les données MAP sont synchronisées au début de chaque heure.
