Brightcove Campaign fournira un ensemble de données de visionnage de vidéos à des champs spécifiques de votre plate-forme intégrée. Vous trouverez ci-dessous un exemple de ces champs et une description des données qui rempliront chaque champ.

Champs standard

Ces champs seront toujours renseignés par Brightcove Campaign.

terrain Description
video_id

ID de la vidéo Brightcove associée à l'événement de visionnage
nom_vidéo

Le nom de la vidéo dans Video Cloud
tracking_id Vous pouvez ajouter un identifiant de suivi personnalisé à une URL
external_id Le GUID (Global User ID) utilisé pour identifier l'utilisateur
player_id ID du lecteur Brightcove qui a créé l'événement View
page_url L'URL de la page sur laquelle la vidéo a été visionnée
regardait Pourcentage regardé: jusqu'où ils ont regardé la chronologie
time_watched Secondes de la vidéo regardée: le temps total qu'ils ont réellement passé sur cette vidéo
created_at Date de création
updated_at Dernière mise à jour
is_synced Valeur booléenne indiquant si l'événement view a été synchronisé ou non avec la plate-forme tierce.

Champs supplémentaires

Ces champs ne sont pas renseignés automatiquement. Ils sont utilisés pour suivre des données personnalisées supplémentaires. Par défaut, ils sont configurés pour capturer l'interactivité des expériences in-page Gallery. Si vous n'utilisez pas l'interactivité, ces champs seront vides.

terrain Description
event_1, event_2, event_3

Événements personnalisés
metric_1, metric_2, metric_3

Métriques personnalisées
Champs UTM - UTMSource, UTMMedium, UTMCampaign, UTMTerm et UTMContent Pour plus d'informations sur l'utilisation des champs UTM, voir Utilisation des champs UTM dans le module Campaign.

Suivi des champs en option

Vous pouvez utiliser cette fonctionnalité si vous souhaitez envoyer des données supplémentaires à un MAP tiers sans utiliser de JavaScript supplémentaire pour tirer parti de l'API côté client de Campaign pour un suivi personnalisé des métriques.

Une fois activé, chaque appel d'événement enregistré à partir d'un lecteur compatible avec la campagne inclura également les paires clé/valeur pour les métadonnées vidéo demandées.

Mises en garde importantes

  • Cette fonctionnalité n'est actuellement prise en charge que pour les intégrations Eloqua (il est possible que d'autres intégrations tierces soient prises en charge à l'avenir)
  • Cette fonctionnalité ne peut être utilisée que pour suivre les types de métadonnées suivants:
    • ID de référence vidéo
    • Une liste de balises vidéo délimitées par des virgules
    • Champs de métadonnées personnalisés des vidéos
  • Jusqu'à 3 valeurs peuvent être définies pour ce type de suivi
  • Une fois cette configuration terminée, Brightcove n'est pas en mesure de supprimer ou de modifier les objets personnalisés envoyés à votre MAP tiers. En d'autres termes, il s'agit d'une configuration unique qui ne peut pas être inversée pour le moment.
  • Ces données ne seront pas disponibles via le module Campaign
  • Ces données ne seront pas disponibles via l'API Audience
  • Ces données apparaissent dans les rapports View Events exportés

Comment demander cette fonctionnalité

Pour configurer cela, veuillez contacter le support Brightcove et demander que vous souhaitiez « activer une intégration MAP pour le suivi des champs en option-fr/. Veuillez passer en revue les mises en garde ci-dessus et inclure les éléments suivants dans votre demande:

  • Votre identifiant de compte Brightcove
  • L'ID de l'intégration de données Eloqua sur laquelle vous souhaitez activer cette fonctionnalité
    • Les champs facultatifs sont configurés intégration par intégration. Comme il est possible d'avoir plusieurs intégrations sur votre compte, Brightcove devra connaître l'ID de l'intégration sur laquelle elle doit être activée.
  • Vous souhaitez activer jusqu'à 3 champs. Il doit s'agir d'un ou d'une combinaison des éléments suivants:
    • ID de référence
    • Balises
    • Nom interne d'un champ de métadonnées personnalisées de vidéo existant sur votre compte

Une fois que vous avez transmis cette demande au service clientèle, aucun autre travail n'est requis de votre part pour l'activer dans Eloqua. Une fois que le support client a confirmé que les champs facultatifs ont été activés et que View Events a été synchronisé, vous devriez commencer à voir ces champs supplémentaires être automatiquement ajoutés à votre objet personnalisé BrightCoveVideoView dans Eloqua.
