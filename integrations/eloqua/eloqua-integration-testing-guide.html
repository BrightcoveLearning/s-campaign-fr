Lors de la mise en place d'une nouvelle intégration Eloqua, il peut être difficile de tester et de confirmer que les données de visionnage vidéo sont bien transmises à Eloqua. Dans cette rubrique, vous apprendrez comment tester et confirmer systématiquement que l'intégration fonctionne et que les données de visualisation et les prospects sont envoyés à Eloqua.

Remarque: Pour plus d'informations sur la configuration du module Campaign pour une utilisation avec Eloqua, voir Intégration de Video Cloud à Oracle Eloqua.

Avant de commencer

Avant de valider l'intégration, on suppose que vous avez:

  • Intégration réussie de Brightcove Campaign avec Eloqua
  • Création d'au moins un lecteur compatible avec la campagne et configuration pour utiliser la connexion Eloqua

Configuration d'un formulaire de pistes de test et publication d'une vidéo

À des fins de test, vous devrez créer et configurer un formulaire de prospect personnalisé dans Campaign. Il s'agit d'un formulaire Eloqua dont vous avez collé le code intégré dans Campaign. Même si vous n'utilisez pas de formulaires de prospects dans votre environnement de production, ce formulaire est utilisé pour être sûr que vous êtes bien connu dans votre base de données et sur la page. Pour créer un formulaire Eloqua et configurer Campaign pour l'utiliser, procédez comme suit.

  1. Connectez-vous à votre compte Eloqua.
  2. Créez et personnalisez un formulaire Eloqua. Pour plus d'informations sur la création d'un formulaire Eloqua, voir Création de formulaires de prospects personnalisés pour Oracle Eloqua.
  3. Cliquez sur Actions > Afficher le HTML du formulaire en haut de la page.
  4. Copiez le code intégré du formulaire dans le presse-papiers.
  5. Revenez au module Campaign.
  6. Créez un nouveau formulaire de prospect à l'aide du code intégré du formulaire Eloqua.
  7. Définissez votre formulaire personnalisé pour qu'il affiche le formulaire au bout de 5 secondes. De cette façon, vous saurez exactement quand il doit apparaître.
  8. Cochez la case pour Toujours afficher le formulaire du prospect, même s'il est déjà connu. Cela permet de s'assurer que le formulaire continue de s'afficher après l'avoir rempli pendant le test.
  9. Enregistrez le formulaire.
  10. Configurez le lecteur Brightcove avec la connexion Eloqua pour utiliser le formulaire de prospects Eloqua qui vient d'être créé.

Générer un lien d'aperçu pour visionner la vidéo

Pour générer un lien d'aperçu vers la vidéo, procédez comme suit.

  1. Ouvrez le module Médias.
  2. Sélectionnez une vidéo et publiez-la sur un lecteur Web. Assurez-vous de sélectionner le lecteur compatible avec la campagne avec le formulaire de prospect.
  3. Cliquez sur le lien d'aperçu pour ouvrir la vidéo dans une nouvelle fenêtre de navigateur. Les tests de cette manière éliminent toutes les variables potentielles qui pourraient être introduites par les lecteurs intégrés à une page HTML.
  4. Vérifiez que la vidéo est lue et que le formulaire de prospect s'affiche au bout de 5 secondes.
  5. Remplissez le formulaire de prospect en utilisant des informations de contact et une adresse e-mail uniques (c'est-à-dire un nom de célébrité ou un nom contenant les mots â€œbctestâ€) afin qu'il puisse être facilement retrouvé plus tard dans Eloqua. Cela élimine toutes les variables de niveau navigateur, lecteur ou base de données. En remplissant le formulaire en tant que nouvel utilisateur, nous pouvons être sûrs que vous êtes désormais un utilisateur connu d'Eloqua.

Confirmer que Campaign a réussi à capturer les données

Avant de rechercher les données de visionnage d'une vidéo dans Eloqua, vérifiez d'abord que Campaign a bien capturé la vue vidéo. Notez que l'affichage des données d'affichage dans l'interface utilisateur de la campagne peut prendre jusqu'à 3 heures. Pour confirmer que Campaign a réussi à capturer les données, procédez comme suit.

  1. Ouvrez le module Campaign.
  2. Cliquez sur Toutes les activités récentes dans le volet de navigation de gauche.
  3. Vérifiez qu'il existe un événement avec un horodatage récent indiquant le nom de la vidéo qui vient d'être visionnée. Le statut de l'événement ne doit pas être traité. Cela indique que Campaign a capturé la vue mais n'a pas encore tenté de la synchroniser avec Eloqua.
  4. Pour déclencher manuellement une synchronisation entre Campaign et Eloqua, cliquez sur Activité de synchronisation dans le volet de navigation de gauche.
  5. Dans la liste déroulante Sélectionner une connexion, sélectionnez votre intégration Eloqua. Un bouton Tester la synchronisation doit apparaître.
  6. Cliquez sur Tester la synchronisation pour déclencher une synchronisation manuelle. Notez que vous êtes limité à la fréquence à laquelle vous pouvez déclencher une synchronisation manuelle. Par défaut, Campaign synchronise les vues avec Eloqua une fois par heure.
  7. Cliquez sur Toutes les activités récentes dans le volet de navigation de gauche.
  8. Le statut de votre événement View sera mis en file d'attente. La synchronisation des données avec Eloqua peut prendre environ 15 minutes. Une fois les données synchronisées, l'état passe à l'un des deux statuts suivants:
    • synchronisé - Indique que l'utilisateur était un utilisateur connu et que les données ont été correctement transmises à son journal d'activité. Les données devraient également être visibles dans Eloqua en quelques minutes.
    • unknown_contact - Indique que l'utilisateur était considéré comme inconnu et qu'Eloqua n'a pas pu accepter les données. Si vous obtenez le statut unknown_contact, vérifiez que vous avez publié la vidéo en utilisant le bon lecteur, que vous avez un formulaire Eloqua en place et que vous testez à l'aide du lien d'aperçu, et non intégré à une page.

Confirmer que les données de visionnage de la vidéo sont dans Eloqua

Une fois que Campaign affiche un statut synchronisé, suivez ces étapes pour vérifier que les données se trouvent dans Eloqua.

Remarque: La mise à jour de l'interface utilisateur d'Eloqua avec les nouvelles données reçues de Campaign prend 5 à 10 minutes.
  1. Connectez-vous à votre compte Eloqua.
  2. Accédez aux objets personnalisés.
  3. Localisez l'objet personnalisé BrightCoveVideoView.
  4. Cliquez sur la flèche de liste déroulante à côté de celui-ci, puis cliquez sur Enregistrements d'objets personnalisés mappés à la répartition des contacts. Une liste des contacts disposant de données de visionnage de vidéos dans cet objet personnalisé s'affiche.
  5. Localisez le nom ou l'adresse e-mail unique que vous avez utilisé lorsque vous remplissez le formulaire de prospect.
  6. Cliquez sur le contact pour ouvrir la page de détails. Une liste de tous les enregistrements de visionnage vidéo du contact s'affiche. Il s'agit des données de visionnage de vidéos transmises par Campaign à Eloqua. Des champs tels que le nom de la vidéo, l'identifiant de la vidéo, l'identifiant du lecteur , le temps de visionnage, le pourcentage de visionnage et

Eloqua exige que les données livrées dans un objet de données personnalisé soient attribuées à un contact connu. Eloqua n'acceptera que les données des contacts connus. Les données des contacts inconnus ne seront pas synchronisées. Si vous n'utilisez pas de formulaires de prospects, il n'y a aucun moyen pour un utilisateur inconnu de se convertir en utilisateur connu et aucune donnée ne sera donc synchronisée.
